## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local TCP clients → HTTP/HTTPS listeners | Untrusted developer/browser traffic enters the daemon over privileged ports. |
| TLS SNI → certificate selector | The requested hostname influences which certificate is presented before HTTP handling begins. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | D | `internal/daemon/network.go` | mitigate | Record bind failures through `SetHTTPBindResult` / `SetHTTPSBindResult`, close partial listeners on startup failure, and keep health truthful instead of reporting bound listeners that never started. |
| T-02-08-02 | S | `internal/proxy/https.go` | mitigate | Keep the managed-suffix gate from `dns.LookupHostname` so only managed hosts can claim local certificates; unmanaged SNI continues to fail locally. |
| T-02-08-03 | R | `internal/proxy/https_test.go` | mitigate | Add regression tests for managed no-route TLS selection so future handshake changes cannot silently break friendly paused/no-route behavior again. |
</threat_model>
