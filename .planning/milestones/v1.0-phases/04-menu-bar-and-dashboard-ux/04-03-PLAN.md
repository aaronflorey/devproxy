<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systray click events -> action dispatcher | User-triggered menu events invoke local admin actions and browser opens |
| menubar runtime -> admin API | Unprivileged UI process requests daemon state and mutations |
| menubar runtime -> `open` command | UI-projected URLs are passed to a macOS process launcher |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | E | start-at-login toggle path | mitigate | Restrict toggle actions to the daemon-approved `menubar` role and surface daemon-role state as read-only status text. |
| T-04-10 | T | route open action | mitigate | Launch only the daemon-provided `OpenURL`; reject empty or non-http(s) URLs before calling `open`. |
| T-04-11 | D | polling loop | mitigate | Use bounded polling intervals and menu refresh paths; avoid spawning overlapping admin requests or browser commands on repeated clicks. |
| T-04-12 | I | offline/degraded menu text | mitigate | Show explicit admin/daemon failure text instead of stale healthy state so users do not act on misleading routing status. |
</threat_model>
