<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| menubar/dashboard -> admin API | Untrusted local UI inputs cross into daemon control handlers |
| admin API -> launchd helpers | UI-triggered startup toggles reach privileged service-management logic |
| route projection -> browser open action | Daemon-projected URLs control which local target the UI opens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | T | `internal/adminapi/server.go` | mitigate | Accept only fixed enum roles (`daemon`, `menubar`) and fixed control endpoints; reject arbitrary launchd labels/paths from request bodies. |
| T-04-02 | I | `internal/admin/routes.go` | mitigate | Build `OpenURL` from authoritative snapshot/runtime metadata only; never accept UI-supplied URLs or hostnames. |
| T-04-03 | D | `internal/adminapi/server.go` | mitigate | Keep the UNIX socket local-only with existing `0600` permissions and return bounded issue/log payload sizes. |
| T-04-04 | R | startup toggle operations | mitigate | Record startup-toggle failures in the session issue buffer with role, action, and error text for operator traceability. |
</threat_model>
