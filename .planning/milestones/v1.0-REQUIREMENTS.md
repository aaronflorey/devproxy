# Milestone v1.0 Requirements Archive

**Status:** Shipped 2026-05-12
**Requirements Satisfied:** 34/34

## Scope

This file snapshots the shipped `v1.0` requirement set that was fully satisfied at milestone closeout.

## Discovery And Routing

- [x] `DISC-01` Discover running containers with published TCP ports on startup and refresh.
- [x] `DISC-02` Keep route state current as containers start, stop, die, destroy, rename, and update.
- [x] `DISC-03` Prefer Docker Compose labels, with container-name parsing only as a fallback.
- [x] `DISC-04` Route only eligible running containers with usable published TCP ports.
- [x] `DISC-05` Resolve domain conflicts deterministically with priority and stable tie-breakers.
- [x] `DISC-06` Surface conflict warnings consistently in logs, `doctor`, `status`, and the dashboard.

## Domains And Overrides

- [x] `DOMN-01` Default service domain `{service}.{project}.{suffix}`.
- [x] `DOMN-02` Root-service mapping at `{project}.{suffix}`.
- [x] `DOMN-03` Laravel Sail-friendly root and companion subdomain behavior.
- [x] `DOMN-04` Docker label overrides for enable/disable, domains, root mapping, port, scheme, and priority.
- [x] `DOMN-05` Config overrides with Docker labels taking precedence for overlapping fields.
- [x] `DOMN-06` Explicit local-only custom domains with DNS warnings for unmanaged suffixes.
- [x] `DOMN-07` Invalid label fields ignored with explicit warnings.

## DNS Proxy And HTTPS

- [x] `NET-01` Managed suffix resolves to `127.0.0.1`.
- [x] `NET-02` HTTP requests proxy to the selected localhost published port with the correct upstream scheme.
- [x] `NET-03` HTTPS uses locally trusted `mkcert` certificates.
- [x] `NET-04` Certificates regenerate when served hostnames change and reuse valid certs when unchanged.
- [x] `NET-05` HTTP on `80` and HTTPS on `443`, with redirect disabled by default.
- [x] `NET-06` WebSocket traffic works through devproxy.
- [x] `NET-07` Managed hostnames without active mappings return a friendly no-route response.
- [x] `NET-08` Paused routing preserves DNS resolution and returns a friendly paused response.

## Install Lifecycle And Diagnostics

- [x] `OPS-01` `devproxy install` creates required paths, resolver state, certificates, and daemon service.
- [x] `OPS-02` Menu bar auto-start installs only with `devproxy install --with-menubar`.
- [x] `OPS-03` Foreground daemon startup fails explicitly when Docker, cert prerequisites, or listeners are unavailable.
- [x] `OPS-04` `devproxy status` exposes daemon health and route counts.
- [x] `OPS-05` `devproxy routes` and `devproxy refresh` inspect and refresh live mappings.
- [x] `OPS-06` `devproxy doctor` checks Docker, DNS, resolver, listeners, mkcert, CA, launchd, proxy reachability, and example domain resolution.
- [x] `OPS-07` `devproxy logs` streams current-session daemon logs.
- [x] `OPS-08` `devproxy uninstall` supports selective cleanup of config, state, logs, and certificates.
- [x] `OPS-09` CLI, dashboard, and menu bar share one local admin API socket source.

## Menu Bar And Dashboard

- [x] `UI-01` Menu bar shows daemon health and active routes.
- [x] `UI-02` Menu bar supports refresh, dashboard, logs, doctor, pause/resume, and start-at-login actions.
- [x] `UI-03` Menu bar opens routes with HTTPS when ready, otherwise HTTP fallback.
- [x] `UI-04` Dashboard shows health, active routes, recent conflicts, and current-session errors.
