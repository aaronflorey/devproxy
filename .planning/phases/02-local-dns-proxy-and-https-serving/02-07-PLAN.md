<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Snapshot hostname inventory → certificate planner | Winning route hostnames determine which SANs will be trusted by the local HTTPS listener. |
| Runtime assembly → external `mkcert` process | Daemon code hands SAN lists and output paths to an external binary that may fail or be missing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07-01 | T | `internal/daemon/network.go` | mitigate | Build inventory only from `routing.Snapshot` winners per D-08 so losing or unmanaged hostnames cannot enter certificate issuance. |
| T-02-07-02 | D | `internal/daemon/network.go` + `mkcert` | mitigate | Surface issuance errors directly from `IssueCertificate` and fail runtime creation instead of reporting false-ready HTTPS state. |
| T-02-07-03 | I | prepared certificate cache paths | mitigate | Reuse stored cert metadata only when SAN coverage is verified by `certs.BuildCertificateInventory`, preventing stale cert reuse after hostname-shape changes. |
</threat_model>
