# Security Policy

## Supported Versions

Security fixes are applied on the default branch and included in the next tagged release.

## Reporting a Vulnerability

Do not open public issues for security reports.

Use GitHub private vulnerability reporting for this repository when it is enabled. If private reporting is unavailable, contact the repository owner directly through GitHub before disclosing details publicly.

Please include:

- affected version or commit
- reproduction steps
- impact summary
- any suggested mitigation
